# GitLab CI Pipelines

Generates reusable GitLab CI pipeline files for opted-in managed service repositories.

Supported pipelines:

- `build_pipeline`, which writes `ci-pipelines/build.gitlab-ci.yml` on
  `feature/build-gitlab-ci-template` and opens a merge request titled
  `Add reusable build-only GitLab CI template`
- `deploy_pipeline`, which writes `ci-pipelines/deploy.gitlab-ci.yml` on
  `feature/deploy-gitlab-ci-template` and opens a merge request titled
  `Add reusable Helm deploy GitLab CI template`

Both pipeline families are generated by default for managed projects when the
per-project config blocks are omitted. Set `build_pipeline = null` or
`deploy_pipeline = null` on a project to opt out of a generated file.

Root `.gitlab-ci.yml` files are not modified. The generated merge request
description tells the operator to add a local include manually and define
concrete jobs through `extends: .build` or `extends: .deploy` with CI variables
only.
